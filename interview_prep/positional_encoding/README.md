# Positional Encoding (Позиционные кодировки)

## Зачем они нужны?

Self-attention — permutation invariant операция:
- score(q_i, k_j) зависит только от содержимого q_i и k_j, не от их позиций
- Если переставить токены — attention weights изменятся соответственно, но модель "не заметит" что порядок изменился
- Вывод: без PE, self-attention = weighted bag-of-words

---

## Виды позиционных кодировок

### 1. Sinusoidal (Абсолютные, Vaswani et al. 2017)
```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```
- **Низкие i** (размерности 0-7): высокая частота, быстро меняются → кодируют близкие позиции
- **Высокие i**: низкая частота → кодируют глобальные позиции
- Свойство: PE(pos+k) = линейная функция PE(pos) → модель может обобщаться на сдвиги
- Не обучаемые параметры
- Экстраполяция за пределы train длины — плохо

### 2. Learned Absolute (GPT, BERT-base)
- Embedding таблица [max_len, d_model], обучается как любые другие параметры
- Простая, но не обобщается за max_len

### 3. Relative Position Bias (T5, Shaw et al.)
- Вместо добавления PE к эмбеддингам — добавляют bias к attention score
- bias = f(|i-j|) — зависит от разности позиций
- T5: learnable bias, bucketed (многие разности → один bucket)

### 4. RoPE (Rotary Position Embedding, Su et al. 2021)
**Идея:** Умножаем Q и K на комплексную экспоненту, зависящую от позиции:
```
q'_m = q_m * e^(iθ_m)   где θ_m = m * Θ (вектор частот)
k'_n = k_n * e^(iθ_n)

dot(q'_m, k'_n) = Re[q_m · conj(k_n)] * cos((m-n)θ) + ...
```
**Ключевое свойство:** attention score зависит только от (m-n) — **только от относительной позиции**

Практически: пары измерений (x_{2i}, x_{2i+1}) вращаются на угол θ_i * pos:
```
θ_i = 1 / (base^(2i/d))   где base = 10000 обычно
```

**Преимущества RoPE:**
- Относительная позиционная информация embedded в dot product
- Хорошо экстраполирует (лучше абсолютных)
- Легко масштабировать контекст

### 5. ALiBi (Press et al. 2021)
- Добавляет линейный bias к attention score: bias = -slope * |i-j|
- slope разный для каждой головы (фиксированный набор)
- Разработан для экстраполяции: обученный на 1K может работать на 2K
- Не требует изменений при длинном контексте

## Context Length Extension

### NTK-aware Scaling
Проблема: RoPE обучена на L_train позициях, нужно инферить на L_new > L_train

**NTK scaling:** масштабируем базу RoPE:
```
base_new = base * (L_new / L_train)^(d / (d-2))
```
- Без fine-tuning, но теряем локальную чувствительность
- Dynamic NTK: применяем scaling только когда seq_len > L_train

### YaRN (Yet Another RoPE extensioN, Peng et al. 2023)
Ключевая идея: разные частоты RoPE нуждаются в разном масштабировании:
- **Высокие частоты** (малые i, θ_i большой): не нужно масштабировать (уже в диапазоне)
- **Низкие частоты** (большие i, θ_i маленький): нужна интерполяция
- **Средние частоты**: плавный переход

Параметры:
- α (alpha): порог high-frequency → нет scaling
- β (beta): порог low-frequency → полная интерполяция
- Нелинейная функция: `γ(r) = (r/α - 1)/(β/α - 1)` для smooth transition

**Temperature scaling:** при длинном контексте attention score распределение становится более flat → добавляют temperature `t = 0.1 * ln(L_new/L_train) + 1` к softmax

### LongRoPE
- Нелинейный поиск оптимального rescaling factor для каждого измерения (не единый для всех)
- Evolutionary search для нахождения оптимальных параметров

## Типичные вопросы на интервью

### Q: Почему RoPE лучше абсолютных PE?
**A:** RoPE кодирует **относительные** позиции в dot product, что более естественно для attention (важна не абсолютная позиция токена, а расстояние между токенами). Формально: dot(RoPE(q,m), RoPE(k,n)) = f(q, k, m-n).

### Q: Что такое "высокая частота" в контексте RoPE?
**A:** Каждое измерение RoPE имеет свою частоту θ_i. Малые i → большие θ_i → угол быстро меняется с позицией (высокая частота) → кодирует локальные отличия. Большие i → малые θ_i → медленно меняется → кодирует глобальные позиции.

### Q: Как расширить контекст без fine-tuning?
**A:** NTK scaling — увеличить base RoPE так, чтобы угловые диапазоны новых позиций не выходили за пределы обучающих. Dynamic NTK применяет это только когда нужно. YaRN + немного fine-tuning — лучший результат.

### Q: В чём проблема простой линейной интерполяции для context extension?
**A:** При линейной интерполяции все позиции "сжимаются" → близкие токены становятся "ещё ближе" → локальная чувствительность теряется. YaRN решает это, не масштабируя высокочастотные компоненты.

## Частые ошибки
- ❌ Путать sinusoidal PE (добавляются к embedding) и RoPE (умножаются на Q/K)
- ❌ Считать что RoPE абсолютные (нет — они кодируют относительные позиции через вращение)
- ❌ NTK scaling ≠ YaRN: NTK единый скейл для всех частот, YaRN — нелинейный
- ❌ Забывать что без PE attention = permutation invariant
