| Method       | Prompt-level acc.  | Instruction-level acc. |

> SAGE could get stuck in a self-reinforcing loop, where the model repeatedly validates its own (possibly incorrect) reasoning path.

The concern is reasonable in principle. We want to point to Figure 4(b) in the paper, which shows the RM score trajectory across SAGE iterations — it dips before recovering, rather than monotonically increasing. This is evidence against a static self-congratulatory loop, because a frozen self-judge would produce monotonically increasing scores.

The structural reason is that the textual gradient is regenerated fresh each epoch from a newly formed best/worst partition. As new candidates are added to the pool, the partition shifts, and the gradient changes accordingly. The model is therefore not conditioning on the same recommendations repeatedly. We also point to Figure 11 in the Appendix, which shows that response diversity (measured by embedding-space variance) remains non-trivial across iterations — the outputs do not collapse to a single answer. That said, we agree that for very hard problems where every generated candidate is wrong, the gradient degrades to noise. This is exactly the scenario we document in experiment B4 with the small model, and it is an honest limitation we discuss in the paper.
